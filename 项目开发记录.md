# 毕设全栈项目 · 开发记录

> 记录开发过程中用户提出的需求、遇到的问题及主动发现并修复的 Bug，方便复盘与交接。

---

## 一、用户额外需求（功能 & 规范类）

### 1. API 层封装
**来源**：用户反馈  
**描述**：前端接口请求直接写在组件里，可维护性差。  
**处理**：新建 `src/apis/` 目录，按业务拆分为 `auth.ts`、`pages.ts`、`upload.ts`，统一从 `index.ts` 导出，所有组件改为调用 API 方法而非直接使用 `axios`。

---

### 2. 内联样式迁移
**来源**：用户反馈  
**描述**：前端大量使用内联样式（`style={{ ... }}`），难以维护和复用。  
**处理**：为所有页面和组件创建对应 `.module.css` 文件，将静态样式全部迁出，仅保留真正由 props 动态计算的内联样式（如 `height`、`backgroundColor`）。

---

### 3. 防止标题文字被选中
**来源**：用户需求  
**描述**：页面上的大标题、小标题可以被鼠标框选复制，交互体验不好。  
**处理**：在 `global.css` 中为 Ant Design 布局类（`Layout Header/Sider`）、`Typography`、`Menu`、`Button`、`Tag` 等 UI 元素添加 `user-select: none`；各页面 CSS Module 中同步补充 `.header`、`.panelTitle` 等自定义类的选中禁止。

---

### 4. 编辑器离开前保存确认弹窗
**来源**：用户需求  
**描述**：编辑器有未保存内容时，点击返回按钮会直接离开导致修改丢失，无任何提示。  
**处理**：在编辑器页面引入 `isDirty` 状态，点击返回时检测是否有未保存修改，有则弹出 `modal.confirm` 对话框，提供"直接离开（危险）"和"继续编辑"两个选项。

---



---

### 6. 登录鉴权对齐企业标准（Cookie 方案）
**来源**：用户需求  
**描述**：Token 存储在 `localStorage` 中存在 XSS 安全隐患，需要由后端统一管理鉴权。  
**处理**：  
- 后端安装 `cookie-parser`，登录接口将 JWT 写入 `HttpOnly; SameSite=Lax` Cookie，新增 `/auth/logout` 接口清除 Cookie；`auth` 中间件优先从 Cookie 读取 Token，保留 Bearer Header 兼容 API 工具。  
- 前端 `axios` 增加 `withCredentials: true`，删除所有手动注入 `Authorization` 的逻辑；`useUserStore` 去除 `token` 字段，改为持有 `user` 对象；`AppRoot` 启动时静默调用 `getProfile` 恢复会话，期间全屏 Loading，防止路由守卫误跳转。

## 二、用户报告的 Bug

### B-01：tsconfig `ignoreDeprecations` 值无效
**现象**：`tsconfig.json` 中 `"ignoreDeprecations"` 配置报错  
**原因**：TypeScript 升级后该选项值格式变更  
**修复**：移除 `ignoreDeprecations` 及已废弃的 `baseUrl`，`paths` 改为相对路径写法

---

### B-02：找不到模块 `"path"` 类型声明
**现象**：`vite.config.ts` 引入 Node `path` 模块报 TS 错误  
**原因**：缺少 `@types/node`，且 `__dirname` 在 ESM 中不可用  
**修复**：安装 `@types/node`，将 `path.resolve(__dirname, ...)` 替换为 `fileURLToPath(new URL('./src', import.meta.url))`

---

### B-03：MySQL `init.sql` 执行报 ERROR 1064（`???`）
**现象**：PowerShell 通过管道执行 SQL 文件时中文注释变成 `???`，语法报错  
**原因**：PowerShell 默认编码非 UTF-8，管道传输乱码  
**修复**：剥离 SQL 文件中的中文注释，改用 `Get-Content ... | mysql --default-character-set=utf8mb4` 执行

---

### B-04：MySQL 存储中文页面标题变成 `???????`
**现象**：创建/保存页面后，数据库 `pages.title` 字段内容全是问号  
**原因**：`mysql2` 连接池未指定字符集，默认使用 Latin1  
**修复**：在 `backend/src/config/db.js` 连接配置中增加 `charset: 'utf8mb4'`

---

### B-05：Ant Design 静态 API 警告（`message` / `Modal.confirm`）
**现象**：`message.success()`、`Modal.confirm()` 调用时控制台报 `Static function can not consume context`  
**原因**：在 Ant Design v5 中静态方法无法访问 React Context（如动态主题、App 层级配置）  
**修复**：创建 `messageBridge.ts` + `useMessage.ts`，在 `AppRoot` 内通过 `App.useApp()` 获取实例并注入，全局使用实例方法替代静态方法；`Modal.confirm` 改为 `modal.confirm`（来自 `App.useApp()`）

---

### B-06：`structuredClone` DataCloneError（Immer Proxy 无法克隆）
**现象**：编辑器中删除表单旧值时页面崩溃，错误 `Failed to execute 'structuredClone': [object Array] could not be cloned`  
**原因**：Zustand + Immer 的 state 是 Proxy 对象，`structuredClone` 不支持克隆 Proxy  
**修复**：`CanvasItem.tsx` 中将 `structuredClone(component.props)` 替换为 `JSON.parse(JSON.stringify(component.props))`

---

### B-07：Formily `TypeError: 'get' on proxy`（Immer × Formily 冲突）
**现象**：编辑组件属性时控制台报 `'get' on proxy: property '0' is a read-only and non-configurable data property`，触发来自 `common.ts` 的 freeze 逻辑  
**原因**：Formily 内部对 `initialValues` 和 `values` 执行 `freeze`，而来自 Immer store 的 Proxy 对象遇到 freeze 时触发只读保护  
**修复**：`PropsPanel.tsx` 中对 `createForm({ initialValues })` 的入参及 `newForm.values` 均做 JSON 往返序列化（`JSON.parse(JSON.stringify(...))`）再使用

---


---

### B-09：`[antd: Spin] tip only work in nest or fullscreen pattern`
**现象**：编辑器加载时控制台警告，`<Spin tip="加载中...">` 在非嵌套模式下 `tip` 不生效  
**原因**：Ant Design v5 中 `Spin` 的 `tip` prop 仅在 `nested`（包裹内容）或 `fullscreen` 模式下有效  
**修复**：将加载页改为 `<Spin size="large" />` + 独立 `<Text>` 用 `Space` 排列，不再依赖 `tip` prop

---

### B-10：`Maximum update depth exceeded`（React DnD 无限循环）
**现象**：拖拽画布组件排序时报错 `Maximum update depth exceeded`，`moveComponent` 被无限调用  
**原因**：`useDrop` 使用工厂函数（`() => ({ ... })`）时产生陈旧闭包，每次 hover 拿到的 `index` 是旧值，导致 `moveComponent` 不断以错误参数触发更新  
**修复**：`useDrop` 改为直接对象形式（非工厂函数），每次渲染都能拿到最新 `index`；同时增加鼠标越过元素中线才触发排序的判断，防止频繁调用

---

### B-11：选中 Divider 组件时边框显示不完整
**现象**：选中极矮的 Divider 组件时，`outline` 选中框只能看到部分边  
**原因**：`outline` 不计入盒模型，在极薄元素上会被裁切  
**修复**：将选中样式从 `outline: 2px solid` 改为 `box-shadow: 0 0 0 2px #ff4d4f`，box-shadow 不受 overflow 裁切影响

---

### B-12：登录页无限刷新
**现象**：访问 `/login` 页面时浏览器不断刷新，陷入死循环  
**根因 1**：`AppRoot` 启动时调用 `getProfile()` 检查会话，未登录时返回 401；axios 拦截器无条件执行 `window.location.href = '/login'`，触发全页面刷新，再次触发 `getProfile()`，形成死循环  
**根因 2**：`useUserStore` 中 `isLoggedIn` 使用了 getter 写法，Zustand `set()` 内部用 `Object.assign` 合并状态时将 getter 的当前值以静态值复制，导致 `isLoggedIn` 永远是初始值 `false`  
**修复**：401 拦截器改为只有在 store 已有 `user`（即登录态 session 过期）时才强制跳转；移除 `isLoggedIn` getter，路由守卫直接读取 `s.user` 判断登录状态

## 三、主动发现并修复的问题

### P-01：前端 Vite 模板选错（vanilla TS 而非 React TS）
**描述**：初始化时选用了 vanilla TypeScript 模板，缺少 React 相关配置  
**处理**：手动安装 `react@19`、`react-dom@19`、`@vitejs/plugin-react`，配置 `vite.config.ts` 和 `tsconfig.json`

---

### P-02：`@vitejs/plugin-react` 与 Vite 版本不兼容
**描述**：Vite 6 + `@vitejs/plugin-react@4` 启动时报 peer dependency 错误  
**处理**：将 Vite 降级至 v5（稳定兼容版本），保持 `@vitejs/plugin-react@4`

---

### P-03：antd v6 与 @formily/antd v2 的 peer dependency 冲突
**描述**：`@formily/antd` v2 要求 antd v4，与项目使用的 antd v6 不兼容  
**处理**：将 antd 降级至 v5，改用 `@formily/antd-v5`，补装 `react-is`

---

### P-04：antd v5 + React 19 兼容警告
**描述**：控制台持续输出 `antd v5 support React is 16 ~ 18` 警告，影响调试  
**处理**：在 `main.tsx` 的 `ConfigProvider` 中增加 `warning={{ strict: false }}` 以静默兼容性警告，不降级 React 版本

---

### P-05：`ProductList` / `CategoryNav` 数组 props 未做防御
**描述**：这两个组件直接对 props 中的数组调用 `.map()`，当 props 为 `null` / `undefined` 时会崩溃  
**处理**：为所有数组类型 props 增加 `Array.isArray(xxx) ? xxx : []` 默认值保护

---

### P-06：tsconfig.node.json 缺少 `composite` 配置
**描述**：`tsconfig.node.json` 被 `tsconfig.json` 引用但未设置 `"composite": true`，导致 TS Project References 报错  
**处理**：在 `tsconfig.node.json` 的 `compilerOptions` 中补充 `"composite": true`

---

### P-07：后端错误处理不统一，未挂载全局错误中间件
**描述**：早期部分 controller 直接 `res.status(500).send()`，没有走统一格式  
**处理**：所有 controller 改为 `try/catch + next(err)`，`app.js` 末尾挂载 `errorHandler` 中间件，统一返回 `{ code, message, data }` 格式

---

---

### 7. 购物车组件完整实现
**来源**：用户需求  
**描述**：购物车入口仅为一个悬浮图标，无状态统计、无商品明细，不满足业务需要。  
**处理**：
- 新建 `useCartStore`（Zustand）作为购物车全局状态，提供 `add / decrease / clear` 方法，商品以 `name__price` 作为唯一 key 自动合并；
- `ProductList` 的 `+/-` 数量控件接入 store，替换原本地 `useState<quantities>`，实现与购物车实时联动；
- `CartEntry` 完整重写：底部吸底结算栏（购物袋图标 + 总数徽章 + 总价 + 去结算按钮）+ 底部抽屉（商品明细 + 数量控件 + 清空）；
- 属性面板配置项精简为"结算跳转链接"和"按钮颜色"两项。

---

### 8. 编辑器属性面板宽度可拖拽调整
**来源**：用户需求  
**描述**：右侧属性面板宽度固定，商品列表等配置项内容较多时空间不足。  
**处理**：将右侧 `Sider` 替换为自定义 `div`，左边框挂载 `onMouseDown` 拖拽事件，通过 `mousemove / mouseup` 动态修改宽度，初始宽度持久化至 `localStorage`。

## 二、用户报告的 Bug（续）

### B-13：商品列表数量控件遮挡价格
**现象**：商品数量大于 0 时，`— N +` 控件与价格在同一行导致遮挡  
**修复**：重构商品卡片 JSX，将数量控件移至价格下方独立一行，CSS 设置 `justify-content: center` 居中显示

---

### B-14：`CartEntry` 渲染崩溃（`price.toFixed` 报错）
**现象**：点击商品 `+` 按钮后购物车抽屉渲染崩溃，`Cannot read properties of undefined (reading 'toFixed')`  
**原因**：Formily/Immer proxy 传递的商品 props 中 `price` 字段在未填写时为 `undefined`  
**修复**：`useCartStore.add()` 写入时做防御性清洗（`Number(price) || 0`）；`CartEntry` 所有 `toFixed()` 调用前均加 `Number()` 转换

---

### B-15：`CartEntry` 选中时无标签且无法删除
**现象**：在编辑器画布中选中购物车组件，既不显示"购物车入口"类型标签，也无法点击删除按钮  
**原因**：`CartEntry` 底部栏 `z-index: 100` 高于 `CanvasItem` 操作层的 `z-index: 20`，按钮被遮挡；`position: sticky` 在编辑器画布内定位异常  
**修复**：`CanvasItem.module.css` 中将 `.actionBar` 和 `.typeLabel` z-index 提升至 200；为 `CartEntry` 增加 `__editorMode` prop，编辑器画布通过 `CanvasItem` 统一注入，编辑器内改为普通流式布局，预览页保持 sticky

---

### 9. 项目（Project）层架构重构
**来源**：用户需求  
**描述**：平台从"用户直接管理页面"升级为"用户管理项目，项目包含页面"的两级架构，解决底部导航须每页手动配置的问题。  
**处理**：
- **数据库**：清库重建，新增 `projects` 表（含 `navbar_config` JSON 字段），`pages` 表增加 `project_id`、`page_type`（mall/checkout/orders/custom）、`is_default`（锁定标记）字段；
- **后端**：新增 `/api/projects` CRUD，`POST /api/projects` 自动创建三个默认页面（商品页/支付页/订单页）并写入默认 NavBar 配置；`DELETE /api/pages/:id` 对 `is_default=1` 的页面返回 403；`GET /api/pages/:id` 改为公开路由（预览无需登录），响应附带项目 NavBar 上下文；
- **前端**：Dashboard 重构为项目列表；新增 `ProjectDetail` 页展示项目内页面（含锁定标识）；编辑器顶部加面包屑导航；Preview 自动读取 `project.navbar_config`，将 `pageType` 引用解析为实际 `pageId` 后渲染 NavBar（支付页除外）；
- **关键设计**：NavBar 配置存在项目层，items 用 `pageType`（'mall'/'orders'）引用而非硬编码 ID，Preview 运行时动态解析，修改一次全项目生效。

---

### 10. 购物车跨页面持久化
**来源**：配套需求  
**描述**：从商城页跳转到结算页时，Zustand 内存状态因页面刷新丢失。  
**处理**：`useCartStore` 接入 `zustand/middleware/persist`，使用 `sessionStorage` 持久化，关闭标签页自动清空。

- 前端：`OrderConfirm` 组件读取持久化购物车，展示商品明细 + 合计，提交后清空购物车并显示大号取餐号；编辑器内显示 mock 数据且禁用真实提交。

### 22. 预览页 fixed 元素宽度约束
**来源**：用户反馈（NavBar、CartEntry 抽屉占满整个浏览器宽度而非画布宽度）  
**根因**：`position: fixed` 默认相对视口定位，不受 `max-width` 父容器约束。  
**CSS 技巧**：为预览容器 `.inner` 添加 `transform: translateZ(0)`，使其成为所有 `position: fixed` 子元素的包含块（CSS 规范：拥有 transform 的元素会创建新的包含块）。此后 NavBar 的 `left: 0; right: 0` 约束在 `.inner` 宽度内，无需修改任何组件。  
**CartEntry Drawer 额外处理**：Ant Design `Drawer` 默认将内容挂载到 `document.body`（脱离包含块范围），通过 `getContainer={() => document.getElementById('preview-container') ?? document.body}` 将其挂载到 `#preview-container`（即 `.inner`），配合其上的 `transform`，Drawer 的 `position: fixed` 也被约束在预览容器内。  
**同步修改**：
- `Preview.module.css`：`.outer` 改为 `height: 100vh`，`.inner` 改为 `height: 100vh; overflow-y: auto; transform: translateZ(0)`，外部灰色背景形成"手机框"视觉；
- `Preview/index.tsx`：`.inner` 加 `id="preview-container"`；
- `NavBar`：`position: fixed; bottom: 0; left: 0; right: 0`（现已受包含块约束），占位 div 高度补充 `env(safe-area-inset-bottom)`。

### 31. 购物车结算页空购物车 Bug（iOS Safari sessionStorage）
**来源**：手机预览时结算页购物车总是空的  
**根因**：iOS Safari 在 `window.location.href` 跳转时会清空 `sessionStorage`（热点网络 / 隐私模式下尤为明显）。Zustand persist 从空 sessionStorage 恢复后 `items = []`，触发"购物车是空的"逻辑。  
**处理**：`useCartStore` 存储从 `sessionStorage` 改为 `localStorage`，跨页面导航稳定持久；购物车仍由支付成功后的 `clear()` 主动清空，不依赖 storage 生命周期。

---

### 32. 预览二维码显示局域网 IP（而非 localhost）
**来源**：手机扫码扫出 localhost，无法访问  
**处理**：
- 后端新增 `GET /api/local-ip` 接口，用 `os.networkInterfaces()` 枚举非 loopback IPv4 地址并返回；
- Preview 页启动时请求该接口，将 QR 码 URL 中的 `localhost` / `127.0.0.1` 替换为真实局域网 IP；
- 切换网络后只需重启服务，IP 自动更新。

---

### 33. CartEntry 抽屉关闭后仍可滚动看见
**来源**：预览页滚到底部能看见购物车面板内容  
**根因**：`.sheet` 使用 `position: fixed; transform: translateY(100%)` 隐藏，但 `#preview-container` 的 `overflow-y: auto` 不裁剪 fixed 子元素的 translate 溢出区域，滚动可见。  
**处理**：`.sheet` 关闭状态加 `visibility: hidden`，配合 `transition: visibility 0s 0.28s`（等滑出动画结束后才隐藏）；打开时 `transition: visibility 0s` 立即显示。遮罩同理。

---

### 34. 支付返回状态同步机制重构（Preview 单点消费）
**来源**：用户需求（避免在多个组件重复补丁，统一页面刷新/同步策略）  
**根因**：支付流程跨 H5 页面与外链回跳，`sessionStorage` 在 iOS Safari 等场景不稳定；清空购物车逻辑分散在 `OrderConfirm` / `PaymentResult`，导致行为不一致。  
**处理**：
- 新增 `paymentStorage` 一次性标记 `post_payment_sync`，提供 `markPostPaymentSync()` 与 `consumePostPaymentSync()`；
- 支付成功（支付宝与 mock）时只负责“打标记”；
- `Preview` 页面新增统一同步器：在 `mount`、`pageshow`、`visibilitychange(visible)` 时消费标记，命中后执行 `clearCart()`，并立即清除标记（幂等）；
- 移除返回按钮上的分散 `clearCart` 副作用，改为页面入口统一处理，状态流更清晰。

**处理**：

**后续优化（记下不做）**：顾客维度隔离（`customer_id` / openid）、预览 slug 防枚举、商家后台仅能查自有 project。

### 36. CartEntry 抽屉替换为成熟组件（antd-mobile Popup）

**根因**：自定义 mask/sheet 动画需要维护大量边界条件（滚动锁、显示时序、遮罩区域、iOS 行为差异），维护成本高。  
**处理**：

### 38. NavBar 配置同步与回归修复
**来源**：用户验证反馈（激活色异常、新增/删除 Tab 同步异常、运行时报错）  
**根因**：
- `TabBar` 内置状态与编辑器属性面板更新节奏不同步，导致 canvas 观察到“配置已改但未即时反映”；
- `ArrayItems` 增删过程存在瞬时 `null`，旧代码 `findIndex + 非空断言` 触发 `Cannot read properties of null (reading 'pageUrl')`。  
**处理**：
- 渲染策略分层：**预览模式**继续使用 `antd-mobile TabBar`，**编辑器模式**改为静态渲染（纯 props 驱动，确保即时同步）；
- 新增 Tab 默认值（icon/label/pageUrl）并配置 `ArrayItems.Addition.defaultValue`；
- Tab key 改为稳定键（`url:*` / `idx:*`），修复增删后的状态错乱；
- 对 `items` 先做空值清洗，移除高风险非空断言，修复运行时崩溃。

### 42. iOS 预览左右留白修复（方案A临时落地）
**来源**：用户手机验收反馈（iPhone 上预览左右存在明显留白）  
**根因**：预览容器固定 `maxWidth=375`，在 390/393/430 等更宽移动视口中会居中留白。  
**处理（方案A）**：

**后续（方案C）**：长期仍建议做完整响应式体系重构（统一尺寸 token、组件级断点策略），当前先保障交付观感。
